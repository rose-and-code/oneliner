# OneLiner — MVP 产品需求文档

## 一、当前阶段目标

验证用户是否会对「每天阅读几句话」产生稳定期待与复访行为。

## 二、产品核心原则

- **极简** — 没有多余元素
- **高频** — 每天都想打开
- **轻负担** — 几秒钟即可完成

## 三、核心交互结构

| 手势 | 语义 | 优先级 |
| --- | --- | --- |
| ⬆⬇ 上下滑 | 同一本书的不同句子 | 主行为 |
| ⬅➡ 左右滑 | 换书 | 次行为 |
| 点击卡片 | 展开上下文 | — |
| 点击「延展」按钮 | 卡片翻面（解释层） | — |

## 四、信息架构（仅 3 层）

### Layer 1：句子层（默认状态）

**页面元素**

- 大号主句（视觉中心）
- 作者名
- 顶部：书名 · 当前句数 / 总句数
- 底部：章节信息
- 底部为 tab bar，常驻
- 需要有操作示意（让用户知道点击后可展开、有上下左右滑的预期）；示意要跟整体 UI 和谐，不要太突兀也不要太不显眼；
- 整张卡片几乎占满屏幕，类似 tiktok 的感觉，无论卡片内容如何变化，卡片大小不要变化

**行为**

- 上滑 → 下一句
- 下滑 → 上一句
- 左滑 → 下一本书
- 右滑 → 上一本书
- 点击主句 → 展开上下文
- 点击 “收藏” 按钮，收藏到 “我的”；“我的” 中呈现用户所有收藏的卡片；暂不涉及点赞、分享等其他常规社交功能
---

### Layer 2：上下文层（点击展开）

**动效**

- 主句不变、位置不动
- 上下文淡入出现，注意，要上面和下面都有；比主句暗一点，但注意要依然保持清晰可读，不要太淡

**内容**

- 原文上下文 3–6 行
- 主句仍然保持视觉中心
- 收藏按钮

**行为**

- 再次点击主句 → 收起
- 点击下方小按钮「翻一面」 → 进入翻面层

---

### Layer 3：翻面延展层（点击按钮触发）

**动效**

- 卡片翻面（类似翻书页）
- 动效缓慢、有阻尼

**内容（必须控制在一屏内）**

- AI 极短解释（≤80 字）：解释这句话在挑战什么，或指出其思想张力
- 一句对立观点（需要有显著标注出来是对立观点，注意取个有趣的名字）；标明这个反面思想的书籍出处和作者（注意，可以不是原句，但要保证核心思想是正确的）

**行为**

- 再次点击 → 翻回正面

## 五、UI 设计

### 风格：整体具有高级感和趣味性结合，选择风格 C
### 端：以模仿 iPhone 的移动端小程序呈现

## 六、预置数据

### 书籍：《人类简史》《富有创造力的心灵》《健全的社会》《智能简史》《纳瓦尔宝典》《第二性》；每本书 20 句经典金句话；要确保每句话必须出自这本书，不得编造
### 使用数据库，不要硬编码
